# R223W v0.1 / v0.2 difference summary spec

## Policy

The gate must show short difference summaries before detailed comparison.

## Summary format

Each sample should show:

```text
v0.1 limitation: one sentence
v0.2 candidate improvement: one sentence
hold risk: one sentence
```

## Required summary axes

- teacher default readability
- classroom event depth
- unit intensity effect
- review ledger completeness
- field leakage risk
- component safety
- sample contamination

## Not allowed

The summary must not become a schema explanation page.

## Hold condition

Hold if reviewers cannot tell whether v0.2 improves teacher output, or if the summary implies v0.2 is already published.
